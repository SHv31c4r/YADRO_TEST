# Привет ревьюверу

### Раздел 1: Работа со скриптом

Скрипт написан на bash. Может запускаться с аргументом `auto` или без него. Если запустится без аргумента, то потребует ручного ввода всех нужных кодов.

Как он работает:

0. Для начала скрипт создает временный файл, куда парсит html для вытягивания текстового описания http-кода;

1. Скрипт пробует вытянуть с главной страницы все доступные http-коды и формирует из них список;

2. После этого он проверяет, сколько кодов он вытянул? Если вытянул больше нуля - использует имеющийся список. Иначе - переходит к плану Б - берет уже заполненный список с заданными кодами;

3. Полученный любым способом список перемешивается при помощи функции shuf, после чего берет первые 5 кодов;

4. Скрипт проходится по всем кодам из списка. Выводит в лог, куда он стучится и результат, полученный в итоге. Если код больше 400 (т.е. получен код ошибки), то он выводит уведомление EXCEPTION, которое так же направляется в поток ошибок;

5. По завершении скрипт стирает временный файл.

### Раздел 2: Работа с Docker

Создан Dockerfile. Из дополнительных зависимостей ставится только curl, так как остальные утилиты уже есть в базовом Ubuntu. После установки производится очистка кеша, копирование файла скрипта и смена прав на нем (для подстраховки, если права вдруг потеряются при копировании). Завершается все выполнением скрипта с параметром `auto`.

Для запуска использовалась команда: `docker build -t test-script-image --no-cache . && docker run -it --name test-run test-script-image`

### Раздел 3: Автоматизация процесса с помощью Ansible

Playbook для данного раздела определяет целевую систему из файла инвентаря. После чего последовательно проходится по ролям:

1. Установка Docker. В данной роли выполняется установка докера и расширения для работы python и Docker. После этого он добавляет пользователя в группу и проверяет статус работы докера;

2. Создание Docker-образа. На данном этапе формируется image с подтягиванием Dockerfile и скрипта с директории `part2`, проверяется доступность заданного имени контейнера и создается контейнер на основе заданного image. Все результаты работы контейнера улетают в registry, из которого будут вытягиваться логи;

3. Вывод логов. На данном этапе осуществляется вывод статуса, с которым завершился контейнер, и его логи;

4. Очистка файлов. Тут просто удаляются скопированные с директории `part2`.

Для запуска использовалась команда: `ansible-playbook playbook.yml --ask-become-pass -i hosts.ini --limit localhost`
